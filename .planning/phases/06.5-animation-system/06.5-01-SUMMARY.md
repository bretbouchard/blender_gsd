## Key Changes

### 1. AnimationConfig Dataclass (types.py)

Camera animation configuration supporting multiple move types:
- **Types**: static, orbit, dolly, crane, push_in, turntable, pan, tilt, truck, rack_focus, custom
- **Properties**: duration, start_frame, easing, loop, angle_range, radius, distance, direction, elevation_range, subject_rotation
- **Serialization**: Full to_dict() and from_dict() support

### 2. MotionPathConfig Dataclass (types.py)

Procedural camera path configuration:
- **Path types**: linear, spline, bezier, arc
- **Look-at modes**: plumb_bob, manual, object, forward
- **Properties**: duration, interpolation, points, preset, closed

### 3. TurntableConfig Dataclass (types.py)

Product showcase rotation configuration:
- **Properties**: enabled, axis, angle_range, duration, start_frame, easing, loop, direction, rotation_speed
- **Use case**: 360-degree product rotation for e-commerce and product videos

### 4. Animation Preset Loaders (preset_loader.py)

New functions added:
- `get_camera_move_preset(name)` - Load camera move preset
- `get_easing_preset(name)` - Load easing curve preset
- `get_turntable_preset(name)` - Load turntable rotation preset
- `list_camera_move_presets()` - List available camera moves
- `list_easing_presets()` - List available easing curves
- `list_turntable_presets()` - List available turntable presets

### 5. Turntable Presets (turntable_presets.yaml)

8 product rotation presets created:
| Preset | Description | Duration | Loop |
|--------|-------------|----------|------|
| product_standard | Standard 360-degree rotation | 96 frames (4s) | Yes |
| product_slow | Slow, elegant rotation | 240 frames (10s) | Yes |
| product_fast | Fast preview rotation | 48 frames (2s) | Yes |
| product_90 | 90-degree turn | 30 frames | No |
| product_180 | 180-degree turn | 60 frames | No |
| product_270 | 270-degree turn | 90 frames | No |
| console_knob | X-axis rocking for console knobs | 72 frames | Yes |
| control_surface | Y-axis subtle rocking | 60 frames | Yes |

## Decisions Made

| Decision | Rationale |
|----------|-----------|
| AnimationConfig.type as string enum | Flexible for future animation types without enum maintenance |
| TurntableConfig defaults loop=True | Product showcase typically requires continuous rotation |
| Separate dataclasses for each animation type | Clean separation of concerns, easier to extend independently |
| Match existing YAML structure (moves:, easing:, turntables:) | Consistent with camera_moves.yaml and easing_curves.yaml patterns |
