# Phase 21.0: Core GP Module - Research

**Researched:** 2026-03-05
**Domain:** Blender Grease Pencil 2D Animation, Non-Photorealistic Rendering (NPR)
**Confidence:** MEDIUM (Blender API verified, some GP v3 specifics need validation)

## Summary

This research covers Phase 21.0: Core GP Module implementation for creating a programmatic Grease Pencil library. The phase involves building `lib/grease_pencil/` with 12 Python modules supporting 2D NPR animation workflows.

**Key Finding:** Grease Pencil has its own modifier stack completely separate from Geometry Nodes. This is CRITICAL - Geometry Nodes cannot process GP data. All procedural effects must use GP-specific modifiers or material/shader node groups.

**Primary recommendation:** Build a type-safe, stage-pipeline architecture using dataclasses for configuration, GP modifiers for effects, and shader node groups for materials. Follow existing patterns from `lib/animation/types.py` and `lib/geometry_nodes/node_builder.py`.

### Pattern 1: Type-Safe Configuration with Dataclasses

**What:** Use dataclasses for all configuration types, matching lib/animation/types.py pattern
**When to use:** All GP configuration objects

**Example:**
```python
# Source: Based on lib/animation/types.py pattern
from dataclasses import dataclass, field
from typing import Optional, List, Dict, Any
from enum import Enum

class StrokeStyle(Enum):
    SOLID = "SOLID"
    TEXTURE = "TEXTURE"

class FillStyle(Enum):
    SOLID = "SOLID"
    GRADIENT = "GRADIENT"
    TEXTURE = "TEXTURE"

class StrokeMode(Enum):
    LINE = "LINE"
    DOTS = "DOTS"
    BOX = "BOX"

@dataclass
class GPStrokeConfig:
    """Configuration for a single GP stroke."""
    id: str
    points: List[tuple]  # List of (x, y, z, pressure, strength)
    stroke_width: float = 5.0
    material_index: int = 0
    use_cyclic: bool = False  # Closed stroke
    seed: Optional[int] = None

    def to_dict(self) -> Dict[str, Any]:
        return {
            'id': self.id,
            'points': self.points,
            'stroke_width': self.stroke_width,
            'material_index': self.material_index,
            'use_cyclic': self.use_cyclic,
            'seed': self.seed,
        }

    @classmethod
    def from_dict(cls, data: Dict[str, Any]) -> 'GPStrokeConfig':
        return cls(**data)

@dataclass
class GPMaterialConfig:
    """Configuration for GP material style."""
    id: str
    stroke_style: StrokeStyle = StrokeStyle.SOLID
    fill_style: FillStyle = FillStyle.SOLID
    stroke_mode: StrokeMode = StrokeMode.LINE
    color: tuple = (0.0, 0.0, 0.0, 1.0)  # RGBA
    fill_color: tuple = (1.0, 1.0, 1.0, 1.0)  # RGBA
    show_stroke: bool = True
    show_fill: bool = True
    use_stroke_holdout: bool = False
    use_fill_holdout: bool = False
    stroke_texture: Optional[str] = None
    fill_texture: Optional[str] = None
    gradient_type: str = "LINEAR"  # For fill_style GRADIENT
    mix_factor: float = 0.5  # For gradient mixing

@dataclass
class GPLayerConfig:
    """Configuration for a GP layer."""
    id: str
    name: str
    opacity: float = 1.0
    blend_mode: str = "REGULAR"  # REGULAR, MULTIPLY, ADD, etc.
    use_lights: bool = False
    use_mask_layer: bool = False
    stroke_configs: List[GPStrokeConfig] = field(default_factory=list)
```

### Pattern 2: Stage Pipeline Functions

**What:** Deterministic transformation functions following GSD Universal Stage Order
**When to use:** All procedural GP generation

**Example:**
```python

def generate_seed_from_params(params: dict) -> int:
    """Generate deterministic seed from parameters."""
    if 'seed' in params:
        return params['seed']
    # Hash sorted params for consistency
    param_str = str(sorted(params.items()))
    return hash(param_str) % (2**31)

def stage_normalize_gp(params: dict) -> dict:
    """
    Stage 1: Normalize and validate GP parameters.
    Ensures all required fields exist with defaults.
    """
    return {
        'stroke_count': params.get('stroke_count', 10),
        'layer_count': params.get('layer_count', 3),
        'seed': generate_seed_from_params(params),
        'stroke_width': params.get('stroke_width', 5.0),
        'material_config': params.get('material_config', {}),
        'output_path': params.get('output_path', ''),
    }

def stage_primary_gp(normalized: dict) -> dict:
    """
    Stage 2: Primary geometry generation.
    Creates core strokes and layers.
    """
    import random
    random.seed(normalized['seed'])

    strokes = []
    for i in range(normalized['stroke_count']):
        stroke = GPStrokeConfig(
            id=f"stroke_{i}",
            points=[(random.random(), random.random(), 0, 1.0, 1.0)],
            stroke_width=normalized['stroke_width'],
            seed=random.randint(0, 2**31),
        )
        strokes.append(stroke)

    return {**normalized, 'strokes': strokes}

    """
    Stage 3: Secondary modifications.
    Add effects, modifiers, styling.
    """
    # Apply effects, noise, etc.
    return primary_result

def stage_detail_gp(secondary_result: dict) -> dict:
    """
    Stage 4: Detail pass.
    Fine-tuning, cleanup, final adjustments.
    """
    return secondary_result

def stage_output_gp(detail_result: dict) -> dict:
```

### Pattern 3: GP Builder Class

**What:** Main builder class following lib/animation/rig_builder.py pattern
**When to use:** Creating GP objects programmatically

**Example:**

```python
# Source: Based on lib/animation/rig_builder.py pattern
class GPBuilder:
    """
    Build Grease Pencil objects from configuration.

    Usage:
        config = GPObjectConfig(...)
        builder = GPBuilder(config)
        gp_object = builder.build("MyDrawing")
    """

    def __init__(self, config: GPObjectConfig):
        self.config = config
        self.gp_object = None
        self.materials: Dict[str, Any] = {}
        self._instance_id = str(uuid.uuid4())[:8]

    def build(self, name: Optional[str] = None) -> Any:
        """
        Build the GP object from config.

        Args:
            name: Optional name for the GP object

        Returns:
            The created Grease Pencil object
        """
        try:
            import bpy
        except ImportError:
            raise ImportError("GPBuilder requires Blender (bpy module)")

        gp_name = name or f"gp_{self.config.id}"

        # Create GP object (Blender 4.3+ v3 API)
        gp_data = bpy.data.grease_pencils.new(gp_name)
        self.gp_object = bpy.data.objects.new(gp_name, gp_data)

        # Link to scene
        bpy.context.collection.objects.link(self.gp_object)
        bpy.context.view_layer.objects.active = self.gp_object

        # Build materials first
        self._build_materials()

        # Build layers
        self._build_layers()

        # Apply modifiers
        self._apply_modifiers()

        return self.gp_object

    def _build_materials(self) -> None:
        """Create GP materials from config."""
        import bpy

        for mat_config in self.config.materials:
            mat = bpy.data.materials.new(mat_config.id)
            mat.use_nodes = True

            # Create GP material settings
            gp_mat = mat.grease_pencil

            gp_mat.show_stroke = mat_config.show_stroke
            gp_mat.show_fill = mat_config.show_fill
            gp_mat.color = mat_config.color
            gp_mat.fill_color = mat_config.fill_color

            # Set stroke style
            gp_mat.mode = mat_config.stroke_mode.value

            self.materials[mat_config.id] = mat

    def _build_layers(self) -> None:
        """Build GP layers from config."""
        import bpy

        for layer_config in self.config.layers:
            layer = self.gp_object.data.layers.new(layer_config.name)
            layer.opacity = layer_config.opacity
            layer.blend_mode = layer_config.blend_mode
            layer.use_lights = layer_config.use_lights

            # Add strokes to layer
            # ... stroke creation code

    def _apply_modifiers(self) -> None:
        """Apply GP modifiers from config."""
        for mod_config in self.config.modifiers:
            mod = self.gp_object.grease_pencil_modifiers.new(
                mod_config.name,
                mod_config.type
            )
            # Apply modifier properties
            for key, value in mod_config.properties.items():
                if hasattr(mod, key):
                    setattr(mod, key, value)
```

### Anti-Patterns to Avoid

- **Using Geometry Nodes for GP data:** GN cannot process GP data - must use GP modifiers
- **Skipping seed parameters:** All procedural generation must be deterministic
- **No mask support:** Every effect must support masking (SLC requirement)
- **Direct API calls without validation:** Always use stage functions with validation
- **Ignoring Blender 4.3+ API changes:** GP v3 has breaking changes from v2

## Don't Hand-Roll

Problems that look simple but have existing solutions:

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Stroke smoothing | Custom algorithm | `GreasePencilSmoothModifier` | GPU-accelerated, well-tested |
| Stroke noise | Perlin noise | `GreasePencilNoiseModifier` | Integrated, real-time |
| Draw-on animation | Frame-by-frame | `GreasePencilBuildModifier` | Automatic timing |
| Material effects | Shader nodes | `MaterialGPencilStyle` + node groups | Native integration |
| Stroke duplication | Loop in Python | `GreasePencilArrayModifier` | Performance |
| Deterministic seeds | `random.random()` | `hash(params) % (2**31)` | Reproducibility |

**Key insight:** GP modifiers are GPU-accelerated and deeply integrated. Custom implementations will always be slower and less compatible.

## Common Pitfalls

### Pitfall 1: Geometry Nodes Confusion

**What goes wrong:** Attempting to use Geometry Nodes to process GP data
**Why it happens:** GN and GP seem similar in Blender's data model
**How to avoid:** GP has its own modifier stack. Always use `grease_pencil_modifiers` not `geometry_nodes_modifiers`
**Warning signs:** `AttributeError: 'Object' has no attribute 'geometry_nodes_modifiers'` when working with GP objects

### Pitfall 2: Material Index Mismatch

**What goes wrong:** Strokes reference non-existent material indices
**Why it happens:** Materials created after strokes, or index tracking errors
**How to avoid:** Always create materials first, then reference by name or track indices carefully
**Warning signs:** Strokes appearing black or invisible, missing materials in viewport

### Pitfall 3: Seed Non-Determinism

**What goes wrong:** Same parameters produce different results across runs
**Why it happens:** Using `random.random()` without seeding, or dict iteration order
**How to avoid:** Always use `random.seed(hash(sorted_params))` pattern, hash sorted dictionaries
**Warning signs:** Regenerating same config gives different stroke positions

### Pitfall 4: Blender 4.3 API Changes

**What goes wrong:** Code written for GP v2 fails in Blender 4.3+
**Why it happens:** GP v3 completely rewrote the API (layers, frames, drawings)
**How to avoid:** Use new API: `bpy.data.grease_pencils.new()`, `gp_data.layers.new()`, avoid old `gp_object.data.layers.active`
**Warning signs:** `AttributeError` on GP objects, missing properties

### Pitfall 5: Layer Frame Timing

**What goes wrong:** Strokes appear on wrong frames or don't animate
**Why it happens:** GP v3 uses drawings linked to frames differently than v2
**How to avoid:** Use proper frame/drawing association API
**Warning signs:** Static drawings when animation expected, missing keyframes

```python
import bpy

def create_gp_material(
    name: str,
    stroke_color: tuple = (0, 0, 0, 1),
    fill_color: tuple = (1, 1, 1, 1),

    gp_mat.show_fill = show_fill
```

### GP Object Creation (Blender 4.3+)

```python
# Source: Blender 4.3+ GP v3 API
import bpy

def create_gp_object(name: str) -> bpy.types.Object:
    """Create a new Grease Pencil object (v3 API)."""
    # Create GP data block
    gp_data = bpy.data.grease_pencils.new(name)

    # Create object with GP data
    gp_obj = bpy.data.objects.new(name, gp_data)

    # Link to collection
    bpy.context.collection.objects.link(gp_obj)

    return gp_obj

def add_gp_layer(
    gp_obj: bpy.types.Object,
    name: str,
    opacity: float = 1.0,
) -> Any:
    """Add a layer to GP object (v3 API)."""
    layer = gp_obj.data.layers.new(name)
    layer.opacity = opacity
    return layer

def add_gp_drawing(
    gp_obj: bpy.types.Object,
    layer_name: str,
    frame_number: int,
) -> Any:
    """Add a drawing at a specific frame (v3 API)."""
    layer = gp_obj.data.layers[layer_name]

    # Create empty drawing
    drawing = gp_obj.data.drawings.new()
    layer.frames.new(frame_number, drawing)

    return drawing
```
