## One-Liner

Added CinematicRenderSettings dataclass, render enums (QualityTier, RenderEngine, DenoiserType), and render preset loaders for quality profiles and pass presets.

## Summary

This plan establishes the foundation for the render system by adding:

1. **CinematicRenderSettings dataclass** - Complete render configuration with:
   - Quality tiers (viewport, draft, preview, production, archive)
   - Engine selection (EEVEE Next, Cycles, Workbench)
   - Resolution and frame settings
   - Denoising and motion blur options
   - Render pass configuration (Z-depth, cryptomatte, normal, combined)
   - Output format and EXR codec settings

2. **Render enums** - Type-safe enumerations:
   - QualityTier: VIEWPORT, DRAFT, PREVIEW, PRODUCTION, ARCHIVE
   - RenderEngine: EEVEE_NEXT, CYCLES, WORKBENCH
   - DenoiserType: OPTIX, OPENIMAGEDENOISE, NONE

3. **Preset loaders** - Configuration file loaders:
   - get_quality_profile(): Load quality presets from quality_profiles.yaml
   - get_pass_preset(): Load render pass presets from pass_presets.yaml
   - list_quality_profiles(): List available quality profiles
   - list_pass_presets(): List available pass presets

## Decisions Made

| Decision | Rationale |
|----------|-----------|
| CinematicRenderSettings as separate class | Extends basic RenderSettings with pass configuration and quality tiers |
| String-based quality_tier field | Matches existing pattern from AnimationConfig.type |
| Default engine EEVEE_NEXT | Modern Blender 4.0+ default, faster iteration |
| Default samples 64 | Balance between quality and speed for preview tier |
